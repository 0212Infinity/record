# 广州天气 AI

[English](./README.md) | [简体中文](./README-zh.md)

一个基于 **广州** 历史 `QWeather` 7 日天气预报快照构建的本地 AI 天气预测演示项目。

这个项目使用轻量级机器学习流程对原始天气预报做**修正预测**，并通过 Flask 本地网页提供可视化展示。当前版本主要预测：

- 每日最高气温
- 每日最低气温
- 每日降水量
- 白天天气类型

当前版本定位为**天气预报修正模型**，不是专业数值天气预报系统的替代品。

## 功能特性

- 只使用本地 `data/` 目录中的历史天气快照
- 自动清洗空文件和异常天气文件
- 将每条预报记录对齐到对应的 `fxDate -> daily[0]` 标签
- 使用回归模型预测温度和降水
- 使用分类模型预测白天天气文本
- 提供本地可视化面板，包含：
  - 未来 7 天 AI 预测
  - 原始预报与 AI 修正结果对比
  - 历史回测指标
  - 数据质量摘要
- 提供 JSON API，方便二次集成或调试

## 数据集说明

数据以 JSON 文件形式保存，目录格式如下：

- 数据路径格式：`data/YYYY/MM/YYYYMMDD.json`
- 城市：广州
- 每个有效文件包含：7 天预报
- 当前数据有效时间范围：`2023-06-22` 到 `2026-05-04`
- 有效快照数：`532`
- 训练时跳过的空文件或异常快照：`25`
- 最终构造出的对齐训练样本数：`3115`

每个有效 JSON 文件通常包含如下字段：

- `tempMax`
- `tempMin`
- `precip`
- `humidity`
- `pressure`
- `cloud`
- `uvIndex`
- `textDay`
- `iconDay`

原始字段说明保留在 [data.md](/D:/ai/record/data.md)。

## 建模方法

每个天气快照都包含 7 天预报。对其中每一条记录：

- 输入特征来自该快照的 `daily[lead]`
- 标签来自文件名等于该条 `fxDate` 的天气快照
- 标签值取对应目标文件中的 `daily[0]`

因此，这个项目本质上是一个实用的**历史预报修正流水线**：

- 回归模型预测 `tempMax`、`tempMin`、`precip`
- 分类模型预测 `textDay`
- 类别特征通过 one-hot 编码处理
- 训练和测试按时间切分，避免未来数据泄漏

## 环境要求

- Windows，或任何可以运行 Python 3.14+ 的环境
- Python 依赖：
  - `pandas`
  - `numpy`
  - `scikit-learn`
  - `flask`
  - `joblib`
  - `matplotlib`

安装依赖：

```bash
python -m pip install -r requirements.txt
```

如果使用当前机器已配置好的 conda Python：

```bash
python.exe -m pip install -r requirements.txt
```

## 快速开始

先训练模型：

```bash
python -m src.train
```

或使用当前环境里的解释器：

```bash
python.exe -m src.train
```

启动本地 Web 服务：

```bash
python app.py
```

然后打开：

```text
http://127.0.0.1:3000
```

如果 `3000` 端口已被占用，`app.py` 会自动尝试后续可用端口。

## API 说明

### `GET /`

返回本地可视化面板页面。

### `GET /api/summary`

返回数据与模型概要信息，包括：

- 数据起止时间
- 最新快照日期
- 有效与异常快照数量
- 对齐后的训练样本数量

### `GET /api/predictions`

返回最新一份快照对应的未来 7 天预测，包括：

- 原始预报值
- AI 修正后的预测值
- 预测的白天天气类型

### `GET /api/backtest`

返回：

- 历史回测指标
- 测试集逐条结果序列

### `GET /api/quality`

返回：

- 文件总数
- 有效与异常快照数量
- 异常文件示例
- 缺失标签数量

### `GET /api/refresh`

刷新 Flask 应用中的模型缓存与面板数据。

## 开发说明

- 当前版本只支持**广州单城市**
- 空文件和异常快照会被自动跳过
- 数据读取包含多编码回退，因为部分文件不是标准 UTF-8
- 使用 `daily[0]` 作为近似真实标签，因此标签存在一定噪声
- 当前版本不会调用任何外部天气 API

## 已知限制

- 这不是一个可直接用于生产的天气预报系统
- 标签来自历史预报快照，不是官方实况观测数据
- 当前数据只覆盖一个城市，历史样本规模有限
- 天气类别预测准确率明显低于温度回归表现
